Developer Field Notes

Git vs GitHub

Git is a local version control tool that tracks changes in my project.

GitHub is a remote platform where I can store my Git repositories online.

Daily Git Loop

Edit files, stage changes, commit snapshots, and push to GitHub.

Git Command Reference

  • git init - Starts a new Git repository.
  • git status - Shows the current state of the working directory and staging area.
  • git add - Stages changes for the next commit.
  • git diff - Shows changes before they are committed.
  • git commit -m "message" - Saves staged changes with a message.
  • git log --oneline - Shows commit history in a short format.
  • git branch - Shows branches in the repository.
  • git push - Sends local commits to GitHub.
  • git pull - Gets the latest changes from GitHub.

Things I Should Not Commit

I should not commit .env files, secrets, node_modules, or .DS_Store. These files can expose private information or make the repository messy.
